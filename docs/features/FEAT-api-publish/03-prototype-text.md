## 页面 P1：API 列表页（管理后台）

### 1. 页面名称与入口

- **名称**：API 列表
- **入口**：管理后台左侧菜单「API 发布管理」→ 默认进入本页；或「API 发布管理」下「API 列表」子菜单。

### 2. 适用角色/权限

- **数据平台管理员**：可查看、筛选、进入发布/详情/编辑/调试/下线。
- **数据/后端开发（发布者）**：同上，无「调用方管理」入口。
- **业务/前端/外部（查看者）**：无权限进入本页（应跳转文档门户或 403）。

### 3. 页面结构（区块划分）

- **顶部操作区**：左侧「发布 API」按钮；右侧可选「文档门户」链接（新开或跳转）。
- **筛选区**：业务域、数据源、标签（多选）、环境、状态（多选）、关键字搜索框、「查询」「重置」按钮。
- **列表区**：表格，列见字段清单；行操作：详情、编辑、调试、下线（已发布时）；支持勾选列（若支持批量下线）。
- **底部分页**：页码、每页条数选择、总数展示。

### 4. 字段清单（列表列）

| 字段名 | 类型 | 默认值 | 校验 | 是否展示 | 是否可编辑 |
|--------|------|--------|------|----------|------------|
| 勾选 | boolean | false | — | 是 | 是（仅已发布可勾选用于批量下线时） |
| API 名称 | string | — | — | 是 | 否 |
| 请求路径 | string | — | — | 是 | 否 |
| 方法 | enum(GET) | — | — | 是 | 否 |
| 环境 | enum(测试,生产) | — | — | 是 | 否 |
| 状态 | enum(草稿,已发布,已下线) | — | — | 是 | 否 |
| 业务域 | string | — | — | 是 | 否 |
| 数据源 / 表 | string | — | — | 是（合并展示如「ds1 / user_info」） | 否 |
| 版本 | string | — | — | 是（如 v1，推荐标识） | 否 |
| 负责人 | string | — | — | 是 | 否 |
| 更新时间 | datetime | — | — | 是 | 否 |
| 操作 | — | — | — | 是（详情、编辑、调试、下线） | — |

### 5. 交互与按钮 enable/disable 规则

- **「发布 API」**：enabled 始终可点（有权限时）；点击进入发布流程步骤1（P2）。
- **「查询」**：enabled 始终可点；点击后按当前筛选条件请求列表，分页重置为第 1 页。
- **「重置」**：enabled 当任一筛选条件非空时可点；点击后清空筛选条件并重新请求第一页。
- **行内「详情」**：enabled 始终可点；跳转 API 详情页（P4）。
- **行内「编辑」**：enabled 当状态为草稿或已发布时可点；已下线后仅可查看不可编辑（或允许编辑后再次发布）。点击跳转编辑（P4 编辑态）。
- **行内「调试」**：enabled 当状态为已发布时可点；草稿/已下线不可调。点击打开在线调试（P5 弹窗或内嵌）。
- **行内「下线」**：enabled 当状态为已发布时可点；点击弹出二次确认；确认后执行下线。若有调用量/调用方提示则弹窗内展示「当前有 N 个调用方 / 近期有调用，确定下线？」。
- **批量「下线」**（若有）：enabled 当至少勾选一行且均为已发布时可点；否则 disabled。点击二次确认后批量下线；部分失败时提示「成功 M 条，失败 K 条」及失败原因摘要。

### 6. 列表规则

- **筛选**：业务域（单选下拉）、数据源（单选下拉）、标签（多选，可选多个）、环境（单选：全部/测试/生产）、状态（多选：草稿/已发布/已下线）、关键字（输入框，模糊匹配名称、描述、路径）。多条件组合取交集。
- **排序**：默认按「更新时间」降序；可选「按名称」「按路径」「按更新时间」排序，升序/降序可切换。
- **分页**：默认每页 **20** 条；每页条数可选 **10 / 20 / 50**，最大 **50**；总条数展示「共 N 条」。
- **批量操作**：本期仅「批量下线」可选；限制单次最多 **50** 条；部分失败时 toast 或弹窗列出失败项及原因。

### 7. 状态

- **加载态**：筛选/翻页后表格区域展示 loading（骨架或 spinner），按钮不重复提交。
- **空态**：无数据或筛选无结果时，表格区展示「暂无 API」或「暂无符合条件的数据」，可提供「发布 API」引导。
- **错误态**：列表接口失败时，表格区展示「加载失败，请重试」+ 重试按钮；toast 提示错误原因。
- **无权限态**：无本页权限时跳转登录或 403 页，不展示列表。
- **成功态**：筛选/保存/下线成功后，列表刷新或 toast「操作成功」。

### 8. 错误提示文案（示例）

- 列表加载失败：「加载失败，请稍后重试」
- 下线失败：「下线失败：{原因}」
- 批量下线部分失败：「成功下线 M 个 API，K 个失败：{简要原因}」

### 9. 可测试验收点（≥8）

1. 有权限用户进入菜单可见 API 列表页，无权限用户进入得 403 或跳转登录。
2. 筛选区选择业务域/数据源/标签/环境/状态后点击查询，列表仅展示符合条件数据。
3. 关键字搜索对名称、描述、路径做模糊匹配，结果正确。
4. 默认按更新时间降序；切换排序字段与方向后列表顺序正确。
5. 分页默认 20 条/页，切换 10/50 后每页条数正确；总条数展示正确。
6. 点击「发布 API」进入发布流程步骤1（选择数据源与表）。
7. 状态为已发布时行内「调试」「下线」可点，草稿/已下线时「调试」不可点。
8. 点击「下线」弹出二次确认；确认后该 API 状态变为已下线，列表刷新。
9. 若存在调用方或调用量，下线确认弹窗中展示影响提示。
10. 列表加载中显示 loading；无数据时显示空态；接口报错时显示错误态与重试。

## 页面 P2：发布 API - 步骤1（选择数据源与表/视图）

### 1. 页面名称与入口

- **名称**：发布 API - 选择数据
- **入口**：API 列表页点击「发布 API」进入；可为向导步骤条第 1 步。

### 2. 适用角色/权限

- 管理员、发布者；查看者无权限进入。

### 3. 页面结构（区块划分）

- **步骤条**（若有）：第 1 步「选择数据」高亮，第 2 步「配置 API」未完成灰显。
- **区块 1 - 选择数据源**：下拉选择数据源（仅展示启用且可用的数据源）；若数据源不可用则提示「该数据源不可用，请先恢复」并禁用下一步。
- **区块 2 - 选择 Schema**（有 schema 时）：下拉或树形选择 schema；无 schema 的数据源不展示本区块。
- **区块 3 - 选择表/视图**：列表或树形展示当前数据源（及已选 schema）下的表与视图；列：名称、类型（表/视图）、注释；单选一行。
- **底部**：「上一步」（若有）、「取消」、「下一步」。「下一步」仅当已选数据源 + 已选表或视图 时 enabled。

### 4. 字段清单

| 字段名 | 类型 | 默认值 | 校验 | 是否展示 | 是否可编辑 |
|--------|------|--------|------|----------|------------|
| 数据源 | 下拉选择 | 空 | 必选，且数据源可用 | 是 | 是 |
| Schema | 下拉/树 | 空 | 有 schema 时必选 | 是（按数据源是否有 schema 显示） | 是 |
| 表/视图 | 列表单选 | 空 | 必选 | 是 | 是 |
| 表/视图 - 名称 | string | — | — | 是 | 否 |
| 表/视图 - 类型 | enum(表,视图) | — | — | 是 | 否 |
| 表/视图 - 注释 | string | — | — | 是 | 否 |

### 5. 交互与按钮 enable/disable 规则

- **数据源下拉**：选择后请求该数据源下 schema 列表（若有）及表/视图列表；若数据源已禁用或连接失败，展示提示并 **禁用「下一步」**。
- **「下一步」**：enabled 仅当「已选数据源 + 已选表或视图」且数据源可用时；否则 disabled。点击进入步骤2（P3），携带 datasource_id、schema（若有）、table_or_view_name、object_type。
- **「取消」**：enabled 始终可点；点击二次确认「确定放弃当前配置？」后返回 API 列表。

### 6. 列表规则（表/视图列表）

- 无分页（若表/视图数量极大可后续加分页）；默认展示当前数据源（及 schema）下全部表与视图。
- 可按名称、类型（表/视图）筛选（可选）；排序默认按名称升序。

### 7. 状态

- **加载态**：选择数据源后加载 schema/表列表时，区块 2/3 展示 loading。
- **空态**：该数据源下无表/视图时展示「暂无表或视图」。
- **错误态**：数据源列表或表列表接口失败时 toast 或区块内提示，并提供重试。
- **无权限态**：无权限进入则 403 或跳转。
- **成功态**：选择完成点击下一步，正确进入步骤2并带参。

### 8. 可测试验收点（≥8）

1. 进入步骤1仅展示已启用且可用的数据源；选择后加载对应 schema（若有）与表/视图列表。
2. 无 schema 数据源不展示 Schema 区块，直接展示表/视图列表。
3. 表与视图均在列表中展示，且类型、注释正确。
4. 未选数据源或未选表/视图时「下一步」disabled；选齐后 enabled。
5. 数据源不可用时提示「数据源不可用，请先恢复」且「下一步」保持 disabled。
6. 点击「取消」弹出确认，确认后返回 API 列表。
7. 点击「下一步」进入步骤2，且 URL 或 state 中携带数据源、schema、表/视图信息。
8. 表/视图列表加载中显示 loading；接口失败时显示错误态并可重试。

## 页面 P3：发布 API - 步骤2（配置方式与内容）

### 1. 页面名称与入口

- **名称**：发布 API - 配置
- **入口**：步骤1点击「下一步」进入；步骤条第 2 步高亮。

### 2. 适用角色/权限

- 管理员、发布者。

### 3. 页面结构（区块划分）

- **配置方式切换**：Tab 或单选「基于表自动生成」「基于表 + 手写 SQL」。
- **方式 A - 基于表**：
  - 条件字段区：表列列表，勾选作为入参的列；每列可设「参数名」「算子」（=、IN、LIKE、BETWEEN 等）。
  - 返回字段区：勾选返回列（或全选）；可编辑参数名（默认列名）。
  - 分页：开关「启用分页」；开启时必填「单页条数上限」（默认 20，最大 100）。
  - 校验规则：至少一个条件或开启分页，否则提交时校验不通过。
- **方式 B - 手写 SQL**：
  - SQL 编辑区：多行文本框；支持 `:paramName` 参数化。
  - 「校验」按钮：校验仅 SELECT、表在白名单内；结果成功/失败提示。
  - 入参映射（可选）：从 SQL 解析出参数名，展示列表可补充类型/必填。
- **公共 - API 元信息**：名称、路径、HTTP 方法（默认 GET）、业务域、标签（多选）、环境（测试/生产）、负责人、描述；限流：QPS 上限或日调用上限（可选）。
- **底部**：「上一步」「保存草稿」「发布」。发布前校验通过且必填项完整时「发布」enabled。

### 4. 字段清单（API 元信息 + 表驱动/SQL）

| 字段名 | 类型 | 默认值 | 校验 | 是否展示 | 是否可编辑 |
|--------|------|--------|------|----------|------------|
| API 名称 | string | 空 | 必填，长度 1–64 | 是 | 是 |
| 请求路径 | string | 空 | 必填，以 / 开头，同环境唯一 | 是 | 是 |
| HTTP 方法 | enum | GET | 固定 GET | 是 | 否（本期） |
| 业务域 | string | 空 | 可选 | 是 | 是 |
| 标签 | string[] | [] | 可选 | 是 | 是 |
| 环境 | enum | 测试 | 必选 | 是 | 是 |
| 负责人 | string | 空 | 可选 | 是 | 是 |
| 描述 | string | 空 | 可选 | 是 | 是 |
| QPS 上限 | number | 空 | 可选，>0 整数 | 是 | 是 |
| 日调用上限 | number | 空 | 可选，>0 整数 | 是 | 是 |
| 条件字段（表驱动） | 见下 | — | 至少一个条件或开启分页 | 是 | 是 |
| 返回字段（表驱动） | 见下 | 至少一列 | — | 是 | 是 |
| 启用分页 | boolean | false | 表驱动时与条件二选一 | 是 | 是 |
| 单页条数上限 | number | 20 | 1–100，启用分页时必填 | 是 | 是 |
| SQL 内容（手写） | string | 空 | 仅 SELECT，表在白名单 | 是 | 是 |

### 5. 交互与按钮 enable/disable 规则

- **「上一步」**：enabled 始终可点；点击返回步骤1，当前步骤2 内容可暂存或不带回（由实现决定）。
- **「保存草稿」**：enabled 当必填项（名称、路径、环境）已填且（表驱动：条件或分页+返回列 / 手写：SQL 通过校验）时；点击后保存为草稿，不同步网关，toast「保存成功」并可选返回列表或留在本页。
- **「发布」**：enabled 条件同保存草稿，且手写 SQL 已通过「校验」；点击后二次确认「确定发布到当前环境？」；确认后调用发布接口并同步网关，成功 toast「发布成功」并跳转详情或列表；失败 toast 错误原因。
- **「校验」（手写 SQL）**：enabled 当 SQL 非空时；点击后请求后端校验；通过则提示「校验通过」，失败提示「仅允许 SELECT」或「表 xxx 不在白名单内」等。

### 6. 状态

- **加载态**：保存/发布请求中按钮 loading、禁用重复点击。
- **空态**：不适用（本页为表单）。
- **错误态**：保存/发布接口失败时 toast 错误文案；校验失败时在 SQL 区或校验按钮旁展示原因。
- **无权限态**：无权限则 403。
- **成功态**：保存草稿/发布成功 toast，并跳转或刷新。

### 7. 可测试验收点（≥8）

1. 切换「基于表」时展示条件字段与返回字段配置区；切换「手写 SQL」时展示 SQL 编辑区与校验按钮。
2. 表驱动：未选任何条件且未开启分页时点击保存/发布，校验不通过并提示「至少添加一个条件或开启分页」。
3. 表驱动：开启分页后单页上限可配置，超过 100 时校验不通过。
4. 手写 SQL：输入含 UPDATE/DELETE 时校验失败，提示仅支持 SELECT。
5. 手写 SQL：访问非白名单表时校验失败，提示「仅允许访问已选表及白名单内对象」。
6. 手写 SQL 校验通过后「发布」才 enabled（或保存时可保存但发布前必须校验通过）。
7. 必填项（名称、路径、环境）未填时「保存草稿」「发布」disabled。
8. 点击「发布」弹出二次确认，确认后请求发布接口；成功跳转详情或列表并 toast。
9. 发布/保存接口失败时 toast 展示服务端返回错误信息。
10. 路径在同一环境下不可与已有 API 重复，否则提交时报错提示。

## 页面 P4：API 详情/编辑页

### 1. 页面名称与入口

- **名称**：API 详情 / 编辑 API
- **入口**：列表页点击「详情」或「编辑」进入；URL 含 API id（及可选 version）。

### 2. 适用角色/权限

- 管理员、发布者：可查看、编辑（草稿/已发布）、下线、调试。查看者无本页权限（走文档门户）。

### 3. 页面结构（区块划分）

- **顶部**：API 名称、状态标签、环境、版本；操作按钮：编辑（草稿/已发布）、调试（已发布时）、下线（已发布时）、返回列表。
- **Tab 或区块**：概览、配置、版本列表。
  - **概览**：名称、路径、方法、环境、状态、业务域、标签、数据源/表、负责人、描述、限流配置、更新时间等只读展示。
  - **配置**：与 P3 一致的配置内容（表驱动或 SQL），可编辑；编辑后「保存」「发布」。
  - **版本列表**：该 API 的版本列表（版本号、是否推荐、状态、创建时间）；操作：设为推荐、下线该版本。
- **编辑态**：与 P3 相同的表单，保存时带 API id 与版本号（乐观锁）；若服务端返回冲突则提示「数据已被他人修改，请刷新后重试」。

### 4. 字段清单（概览只读 + 配置可编辑）

- 与 P3 元信息及配置字段一致；概览页全部只读；配置 Tab 下与 P3 同表驱动/SQL 可编辑。版本列表列：版本号、是否推荐、状态、创建时间。

### 5. 交互与按钮 enable/disable 规则

- **「编辑」**：enabled 当状态为草稿或已发布时；已下线后仅查看不可编辑（或产品允许编辑后再发布）。点击进入配置 Tab 编辑态。
- **「调试」**：enabled 当状态为已发布时；点击打开在线调试（P5）。
- **「下线」**：enabled 当状态为已发布时；点击二次确认（若有调用方/调用量则提示）；确认后下线。
- **「保存」**（编辑态）：enabled 当必填项完整且配置校验通过；保存时带版本号/更新时间做乐观锁；冲突时提示「数据已被他人修改，请刷新后重试」。
- **「发布」**（编辑态）：enabled 同 P3；点击后同步网关并可选生成新版本或覆盖当前版本（产品约定）。

### 6. 状态

- 加载态：进入详情时接口 loading；保存/发布请求中按钮 loading。
- 空态：版本列表无版本时展示「暂无版本」。
- 错误态：详情接口失败或保存冲突时 toast/页面内提示。
- 无权限态：403。
- 成功态：保存/发布/下线成功 toast 并刷新数据。

### 7. 可测试验收点（≥8）

1. 从列表进入详情，概览展示名称、路径、状态、环境、数据源/表等与列表一致。
2. 状态为已发布时「编辑」「调试」「下线」可点；已下线后「编辑」不可点或仅查看。
3. 配置 Tab 下修改后点击保存，乐观锁校验；若他人已改则提示「数据已被他人修改，请刷新后重试」。
4. 版本列表中可设置某版本为推荐；可下线某版本并二次确认。
5. 点击「下线」弹出确认；若有调用量/调用方则弹窗内展示提示。
6. 点击「调试」打开在线调试（P5）并带入当前 API id。
7. 详情加载失败时展示错误态与重试。
8. 返回列表后列表数据与详情一致（状态、更新时间等）。

## 页面 P5：在线调试（弹窗或页内）

### 1. 页面名称与入口

- **名称**：在线调试
- **入口**：API 列表行操作「调试」、API 详情页「调试」；可为抽屉/弹窗或独立页。

### 2. 适用角色/权限

- 管理员、发布者：可调试任意已发布 API。查看者：仅可调试己方有权限的 API（若产品支持）。

### 3. 页面结构（区块划分）

- **环境选择**：单选「测试」「生产」；默认测试；生产可加二次确认或仅管理员可选（产品约定）。
- **入参区**：根据 API 定义动态生成输入框（参数名、类型、必填）；每项：标签、输入框、 placeholder（示例值）。
- **操作**：「发送」按钮。
- **结果区**：HTTP 状态码、响应 Body（JSON 格式化展示）；可选展示「耗时」。超时（如 10s）时提示「请求超时，请检查参数或稍后重试」。

### 4. 字段清单（动态入参）

- 入参由 API 配置生成：参数名、类型（string/number/array 等）、必填、示例值；均为可编辑输入。

### 5. 交互与按钮 enable/disable 规则

- **「发送」**：enabled 当必填入参已填时（可选：全部入参合法格式）；点击后发起请求，按钮进入 loading、禁用重复点击；请求返回或超时后恢复。
- **环境切换**：切换后清空结果区；选「生产」时可弹确认「即将请求生产环境，是否继续？」。

### 6. 状态

- **加载态**：发送请求中，结果区 loading，按钮 disabled。
- **空态**：未发送前结果区为空或占位「点击发送发送请求」。
- **错误态**：请求失败（4xx/5xx 或网络错误）时结果区展示状态码与错误 Body；超时提示「请求超时」。
- **成功态**：2xx 时展示状态码与 JSON 响应；可展示耗时。

### 7. 可测试验收点（≥8）

1. 打开调试后入参区根据 API 定义展示对应输入项，必填项带 *。
2. 必填入参未填时「发送」disabled；填齐后 enabled。
3. 选择环境后点击发送，请求发往对应环境 baseURL + API 路径。
4. 发送后 10s 内返回则展示状态码与 Body；超过 10s 提示超时。
5. 返回 2xx 时结果区展示 JSON 格式化内容；4xx/5xx 展示相应状态码与 Body。
6. 调试请求写入审计（操作人、时间、API、环境、结果摘要）。
7. 生产环境选择时若有二次确认，确认后才发送。
8. 单次返回条数若有限制，结果中超出部分截断或提示（与 PRD F5.2 一致）。
