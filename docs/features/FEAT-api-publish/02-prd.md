# API 发布管理 - 产品需求文档（PRD）

**FEAT 编号**：FEAT-api-publish  
**文档版本**：1.0  
**依赖澄清文档**：`docs/features/FEAT-api-publish/01-clarification.md`

---

## 1. 背景与问题

### 1.1 业务背景

作为中台服务商交付标准化数据中台系统，需同时支持「数据 API」与「业务 API」的发布与治理。本期优先打通**基于数据源与表/视图的 API 发布闭环**，使客户能够：基于已有数据连接选表或视图，一键或通过手写 SQL 发布为 HTTP API，并通过统一目录、文档与在线调试完成对接。

### 1.2 现状与痛点

- 缺少统一的数据 API 发布入口，数据表暴露为接口的路径不清晰。
- 客户不清楚如何基于表暴露接口、入参/出参如何约定、如何传参与鉴权。
- 无统一 API 目录、可检索的对接文档与在线调试能力，联调与上线成本高。

### 1.3 约束

- 底层网关/运行环境依赖现有或将接入的网关组件，本期不自研网关。
- 安全上仅允许 SELECT，表/视图访问范围受白名单约束。
- 与数据源管理模块共用「prod / 非 prod」环境模型。

---

## 2. 目标与成功指标

### 2.1 产品目标

- 提供「选数据源 → 选表/视图 → 发布为 API」的完整路径，支持**基于表自动生成**与**基于表+手写 SQL**两种方式。
- 提供 API 分类、搜索、在线调试与对接文档（含独立文档页、文档门户、OpenAPI 导出），使调用方能正确调用。
- 支持多版本、环境区分（测试/生产）、鉴权与按 API 限流，满足安全与可运维要求。

### 2.2 成功指标（可验证）

| 指标 | 口径 | 目标值 |
|------|------|--------|
| API 列表筛选+分页响应 | P95 响应时间，数据规模约 1000 条 | ≤ 3 秒 |
| 在线调试单次请求 | P95 响应时间（简单 SELECT） | ≤ 10 秒，超时有明确提示 |
| 发布同步 | 从「保存并发布」到网关可调用的端到端时延 | ≤ 30 秒（或与网关 SLA 一致）【待确认】 |
| 审计覆盖 | 新增/编辑/下线/发布状态变更/SQL 变更/调试发起是否落审计 | 100% 落库可查 |
| 安全校验 | 手写 SQL 含非 SELECT 或访问非白名单表时 | 提交被拒绝，有明确错误提示 |

## 3. 非目标（明确不做什么）

- **不自研 API 网关**：不实现网关的流量转发、负载均衡、 TLS 终结等，依赖现有或即将接入的网关/运行时。
- **不支持存储过程/函数、gRPC、GraphQL、WebSocket**：本期仅 HTTP/REST（或 HTTP+JSON）。
- **不做 OAuth2 完整授权流程、按调用方细粒度配额**：鉴权采用 AppKey/Token + 可选 IP 白名单；限流仅按 API 配置，不按调用方配额。
- **不做可视化 JOIN 拖拽、复杂 SQL 编辑器**：不提供语法高亮/补全、执行计划预览等（可后续迭代）。
- **不做 Mock 服务、Postman 集合导出**：可后续迭代。
- **不做多环境扩展**：本期仅「测试 / 生产」两档，不做 dev/test/pre/prod 多环境独立发布。

## 4. 用户与场景（用户故事）

| 角色 | 用户故事 | 验收要点 |
|------|----------|----------|
| 数据平台管理员 | 作为管理员，我希望在管理后台维护 API 台账、发布/下线 API、管理调用方密钥，以便整体可控。 | 可完成 API 全生命周期操作及调用方管理；操作可审计。 |
| 数据/后端开发（发布者） | 作为发布者，我希望基于已有数据源选表或视图，通过「表驱动」或「手写 SQL」发布 API，并设置分类、环境与限流，以便快速暴露数据能力。 | 支持两种发布方式；可选业务域/标签/环境；可配置 QPS 或日上限。 |
| 业务/前端开发（查看者） | 作为调用方，我希望在文档门户查看 API 列表、入参出参说明与鉴权方式，并能在线调试，以便正确对接。 | 文档门户可检索、可查看单 API 文档、可导出 OpenAPI；支持在线填参调试并看到状态码与响应。 |
| 外部客户/合作方 | 作为外部调用方，我希望获得 AppKey 与调用示例，并按文档发起请求，以便集成。 | 文档中写清 URL、Method、Header/参数、示例；鉴权失败时返回 401 及明确错误码。 |

## 5. 功能需求（编号、优先级、描述、验收标准）

### 5.1 数据连接与对象选择

| 编号 | 优先级 | 描述 | 验收标准 |
|------|--------|------|----------|
| F1.1 | P0 | 基于数据源管理模块已有数据源选择连接；若数据源有 schema，先选 schema 再选表/视图；无 schema 则直接选表/视图。 | 在「发布 API」流程中可选出数据源；有 schema 时下拉或树形展示 schema 再展示表/视图列表；无 schema 时直接展示表/视图列表；选择后能带出表/视图名与注释。 |
| F1.2 | P0 | 支持选择「表」与「视图」两种对象类型，列表展示名称、注释等基础信息。 | 表与视图均在列表中可选；列表至少展示名称、注释（若有）；选择后进入下一步配置。 |
| F1.3 | P1 | 当所选数据源不可用（已禁用或连接失败）时，不允许发布并给出明确提示。 | 选择已禁用数据源时提示不可用；在发布/保存时若连接失败则提示「数据源不可用，请先恢复」并阻止发布。 |

### 5.2 基于表发布 API

| 编号 | 优先级 | 描述 | 验收标准 |
|------|--------|------|----------|
| F2.1 | P0 | 支持将「查询条件字段」配置为入参，名称与类型从表结构带出；支持选择「返回字段」为全表或勾选列，返回类型从表结构推导。 | 配置页展示表/视图列列表；可勾选条件列作为入参并展示类型；可勾选返回列（或全选）；入参名可编辑；保存后生成的 API 入参/出参与配置一致。 |
| F2.2 | P0 | 条件支持等值、IN、LIKE、范围等简单算子；安全策略：必须带至少一个条件或开启分页，分页时单页条数有上限（如 100）。 | 每个条件字段可选择算子（=、IN、LIKE、BETWEEN 等）；未选任何条件且未开启分页时提交校验不通过并提示「至少添加一个条件或开启分页」；分页开启时单页上限可配置且不超过约定值（如 100）。 |
| F2.3 | P0 | 支持设置 API 名称、路径、HTTP 方法（GET）、业务域、标签、环境（测试/生产）、负责人等元信息；支持保存为草稿或直接发布。 | 表单包含名称、路径、方法、业务域、标签、环境、负责人；可保存为草稿（不同步网关）或发布（同步网关）；发布后该环境下 API 可被调用。 |

### 5.3 基于表 + 手写 SQL 发布 API

| 编号 | 优先级 | 描述 | 验收标准 |
|------|--------|------|----------|
| F3.1 | P0 | 选表后支持编写 SQL，支持参数化占位（如 `:id`）；仅允许 SELECT；仅允许访问「已选表 + 同库白名单表/视图」。 | 提供 SQL 编辑区；支持 `:paramName` 形式参数；提交前校验：仅包含 SELECT 且不包含 UPDATE/DELETE/DROP 等危险关键字；校验涉及表/视图均在「已选表+白名单」内，否则拒绝并提示「仅允许访问已选表及白名单内对象」。 |
| F3.2 | P0 | 白名单由管理员维护（同库表/视图列表），配置界面可增删白名单项。 | 存在白名单配置入口；可针对某数据源或某 schema 维护允许访问的表/视图列表；手写 SQL 校验依赖该白名单。 |
| F3.3 | P1 | 提交前可执行语法与白名单校验，并展示校验结果。 | 提供「校验」按钮；校验通过提示成功，失败提示具体原因（语法错误或表不在白名单）。 |

### 5.4 API 管理（台账、版本、状态）

| 编号 | 优先级 | 描述 | 验收标准 |
|------|--------|------|----------|
| F4.1 | P0 | 统一 API 台账：列表展示名称、路径、方法、环境、状态（草稿/已发布/已下线）、业务域、数据源/表来源、负责人、更新时间等。 | 列表页展示上述字段；支持分页；点击行可进入详情/编辑。 |
| F4.2 | P0 | 支持按业务域、数据源、标签、环境、状态筛选；支持关键字搜索（名称、描述、路径）。 | 筛选条件变更后列表刷新；关键字搜索对名称、描述、路径进行匹配（模糊或分词由实现决定）；结果正确。 |
| F4.3 | P0 | 支持多版本（如 v1、v2）；可标记「当前推荐版本」；旧版本可保留一段时间后可下线。 | 同一逻辑 API 可存在多个版本（版本号区分）；可设置其中一个为推荐版本；下线时可选下线某版本或整 API；列表/文档可展示版本与推荐标识。 |
| F4.4 | P0 | 下线前若存在调用量或调用方信息，展示影响提示并二次确认；下线后网关返回 410 或友好提示。 | 下线操作前若有调用关系或调用量则弹窗提示；用户确认后执行下线；下线后实际请求该 API 时返回 410 或约定错误码与提示文案。 |
| F4.5 | P1 | 编辑时若存在并发修改，采用乐观锁或版本校验，后保存者提示「数据已被他人修改，请刷新后重试」。 | 保存时带版本号或更新时间；若服务端已更新则拒绝并提示上述文案。 |

### 5.5 在线调试

| 编号 | 优先级 | 描述 | 验收标准 |
|------|--------|------|----------|
| F5.1 | P0 | 在 API 详情或文档页提供在线调试：选择环境、填写入参、发起请求，展示 HTTP 状态码与响应 Body（JSON）。 | 有「调试」入口；可选择环境（测试/生产）；入参根据 API 定义展示输入框；发起请求后展示状态码与响应 Body；超时（如 10s）有明确提示。 |
| F5.2 | P1 | 调试请求带超时与单次返回条数上限，避免拖垮目标库。 | 调试请求有超时配置（如 10s）；若为分页或限行查询，单次返回条数有上限；超出时截断或提示。 |
| F5.3 | P2 | 在线调试的发起记录入审计（操作人、时间、API、环境、结果摘要）。 | 审计日志中可查到调试操作记录及成功/失败摘要。 |

### 5.6 对接文档

| 编号 | 优先级 | 描述 | 验收标准 |
|------|--------|------|----------|
| F6.1 | P0 | 每个 API 提供独立文档页：请求 URL、Method、入参说明（名称、类型、必填、示例）、出参结构、示例、鉴权方式（Header/参数说明）。 | 从 API 配置自动生成文档；包含上述要素；文档与当前发布配置一致（非手写独立维护）。 |
| F6.2 | P0 | 支持导出 OpenAPI 3.0（或等价）文件；可针对单 API 或全量/按分类导出。 | 提供导出入口；导出文件符合 OpenAPI 3.0 规范或约定等价格式；导入第三方工具可识别。 |
| F6.3 | P0 | 提供文档门户：按分类/标签聚合 API 列表，支持跳转到单 API 文档。 | 有独立文档门户入口；可按业务域/标签等筛选 API 列表；点击进入单 API 文档页。 |
| F6.4 | P1 | 文档中注明「以实际 API 为准」；若支持手写补充说明，需在界面提示与自动生成内容区分。 | 文档页有免责或说明文案；若有自定义补充区块，与自动生成区块视觉或标注区分。 |

### 5.7 鉴权与调用方管理

| 编号 | 优先级 | 描述 | 验收标准 |
|------|--------|------|----------|
| F7.1 | P0 | 为调用方分配 AppKey/Token；API 调用时校验密钥；可选配置 IP 白名单。 | 有「调用方」或「应用」管理入口；可创建调用方并生成 AppKey/Token；可配置 IP 白名单；调用时携带密钥（如 Header）；校验通过则放行，否则返回 401 及明确错误码。 |
| F7.2 | P0 | 支持按「调用方」启用/禁用对某 API 的访问。 | 可配置某调用方对某 API 的访问权限（启用/禁用）；禁用后该调用方请求该 API 返回 403 或约定错误码。 |
| F7.3 | P1 | 密钥不明文落库、传输加密；支持密钥重置或禁用。 | 存储为加密或不可逆摘要；传输使用 HTTPS；支持重置密钥或禁用某调用方。 |

### 5.8 限流与防护

| 编号 | 优先级 | 描述 | 验收标准 |
|------|--------|------|----------|
| F8.1 | P0 | 支持按 API 配置 QPS 或日调用上限；超出时返回 429 或友好提示。 | 在 API 配置中有 QPS 或日上限配置项；超出后请求返回 429 或约定错误码与提示；行为与配置一致（可与网关联动实现）。 |

### 5.9 审计与可观测

| 编号 | 优先级 | 描述 | 验收标准 |
|------|--------|------|----------|
| F9.1 | P0 | 记录 API 新增、编辑、删除（下线）、发布/下线状态变更、SQL 或字段映射变更；记录操作人、时间、对象、操作类型、关键摘要。 | 审计日志表或系统中可按人/时间/对象/操作类型查询；上述操作均有记录且关键信息完整。 |
| F9.2 | P1 | 在线调试发起写入审计（操作人、时间、API、结果摘要）。 | 同 F5.3。 |
| F9.3 | P1 | 发布/下线/调用失败有结构化日志与错误码，便于监控与排障。 | 关键路径有日志；错误码统一或可映射；可与现有日志/监控系统对接。 |

### 5.10 前端形态与权限

| 编号 | 优先级 | 描述 | 验收标准 |
|------|--------|------|----------|
| F10.1 | P0 | 管理后台提供「API 发布管理」菜单：列表、发布、编辑、文档、调试、调用方管理。 | 菜单位置与命名符合约定；上述功能均可从该模块入口到达。 |
| F10.2 | P0 | 独立「API 文档门户」入口：面向调用方，只读；支持在线调试（可为登录后或带临时 Token）。 | 文档门户可独立访问；无需发布权限即可查看文档与调试（鉴权策略可配置）；与管理后台权限分离。 |

## 6. 数据与权限

### 6.1 核心实体与字段口径（概要）

| 实体 | 关键字段 | 说明 |
|------|----------|------|
| API | id, name, path, method, env, status, version, datasource_id, table_or_view_name, publish_type(table/sql), config_json, created_at, updated_at, owner_id | 路径在同一环境+版本下唯一；config_json 含入参/出参映射、SQL（若手写）、限流配置等。 |
| API 版本 | api_id, version, is_recommended, status, config_snapshot, created_at | 与 API 一对多；config_snapshot 为发布时配置快照。 |
| 调用方/应用 | id, name, app_key_hash, secret_encrypted, ip_whitelist, status, created_at | 密钥不以明文存储。 |
| API-调用方授权 | api_id, caller_id, enabled | 启用/禁用某调用方对某 API 的访问。 |
| 表/视图白名单 | datasource_id, schema_name, object_name, object_type(table/view) | 手写 SQL 允许访问的对象集合。 |
| 审计日志 | id, operator_id, operation_type, target_type, target_id, detail_summary, created_at | operation_type 如 create_api, update_api, publish, offline, debug 等。 |

### 6.2 权限矩阵（与澄清一致）

| 角色 | 查看列表/详情 | 发布/编辑/下线 API | 配置 SQL/字段映射 | 在线调试 | 查看文档门户 | 管理调用方/密钥 |
|------|----------------|---------------------|--------------------|----------|--------------|------------------|
| 数据平台管理员 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| 数据/后端开发（发布者） | ✓ | ✓ | ✓ | ✓ | ✓ | — |
| 业务/前端/外部（查看者） | — | — | — | ✓（仅己方） | ✓ | —（仅申请/查看己方密钥） |

### 6.3 审计要求

- 必须落审计：API 新增/编辑/删除、发布/下线、SQL 或字段映射变更、在线调试发起。
- 审计字段至少包含：操作人、时间、目标类型与 ID、操作类型、关键摘要（如 API 名称、版本、变更前后关键字段）。
- 调用方密钥的创建/重置/禁用建议落审计。

## 7. 交互与异常（概要）

- **主流程**：选数据源 → 选 schema（若有）→ 选表/视图 → 选择「表驱动」或「手写 SQL」→ 配置入参/出参或 SQL → 填写元信息 → 保存草稿或发布 → 同步网关。
- **列表与筛选**：多条件筛选与关键字搜索；列表分页；点击进入详情/编辑。
- **文档门户**：分类/标签导航 → 单 API 文档 → 在线调试（选环境、填参、看结果）。
- **异常**：数据源不可用、表/视图不存在、SQL 越权或非 SELECT、无条件且无分页、并发编辑、下线有引用、鉴权失败、限流 429、调试超时等，均需明确提示或错误码；详见 01-clarification 2.5。细节交互与文案在文本原型中按页面补充。

## 8. 依赖与风险

### 8.1 依赖

- **数据源管理模块**：依赖其提供的数据源列表、连接信息及「prod/非 prod」环境；选表/视图时可能依赖其元数据或直连元数据接口。
- **网关/API 运行时**：发布时需将 API 配置（路径、方法、上游类型、参数映射、限流等）同步到网关；需约定同步接口与失败重试策略。【待确认：网关产品形态与接口】
- **认证与权限**：管理后台与文档门户的登录、角色与权限若已有统一体系，需对接；否则本期可内置简单角色模型。

### 8.2 风险

- **白名单维护不当**：扩大 SQL 可访问范围，需规范白名单评审与变更流程。
- **网关与本模块配置不一致**：发布失败或未同步会导致「文档有、实际不可用」，需明确发布流程、同步结果校验与告警。
- **文档门户与管理后台分离**：登录与权限体验需统一或明确说明，避免调用方困惑。

## 9. 发布策略

- **灰度**：本期可不做按流量比例的灰度发布；若网关支持按环境（测试/生产）隔离，先发布测试环境，验证后再发布生产。【待确认：是否需按租户或按 API 灰度】
- **回滚**：支持将 API 版本回滚到上一版本（即再次发布上一版本配置）；下线为直接不可用，如需「快速恢复」可再次发布。
- **监控**：发布/下线/同步失败需有明确日志与告警；API 调用 4xx/5xx 率、限流 429 率建议纳入监控大盘；审计日志保留周期与平台统一策略一致。
