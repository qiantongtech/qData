2) 功能规格草案

2.1 目标（in-scope）
- 统一数据源台账：
  - 支持 MySQL、PostgreSQL、Oracle、SQL Server、Hive、Kafka、FTP/SFTP、HTTP API、Doris 等数据源注册与管理。
- 基础 CRUD：
  - 支持数据源新增、编辑、删除（软删），包含基础连接配置与元信息（名称、描述、业务域、责任人、环境、标签等）。
- 启用 / 禁用控制：
  - 通过状态切换控制数据源是否允许被采集任务引用/新建绑定，只使用“启用/禁用”两种状态。
- 连接测试（手动）：
  - 在详情页或表单中手动触发连接测试，实时返回成功/失败与摘要信息，操作可通过日志/审计追溯到操作者与时间。
- 分组 / 分类与筛选：
  - 支持按业务域、环境（prod/非 prod）、数据源类型、标签、状态等多维度分类与筛选。
- 审计与敏感信息保护：
  - 数据源新增/编辑/删除、敏感字段查看/导出行为有审计记录；密码/Token/密钥等敏感字段采用加密存储与前端脱敏展示。

2.2 非目标（out-of-scope）
- 不做复杂权限体系：
  - 本期不做模块内细粒度权限控制，仅控制少量管理员可登录并拥有全部操作权限。
- 不做自动化健康检查 / 心跳任务：
  - 不支持周期性健康检查、自动探活、对接调度前置检查等自动化机制。
- 不做高级数据预览 / 数据采样：
  - 不在本模块提供通用数据浏览器或数据采样查询，仅关注“连接可用性”与配置台账。
- 不做新增数据源类型扩展：
  - OSS/S3、ClickHouse、ES、MongoDB 等云/新类型数据源明确为后续迭代范围。
- 不强制对接外部 CMDB / 资产系统：
  - 本期以内建数据源台账为主，最多提供导出/接口以供后续集成，不做强绑定。

2.3 用户角色与权限（权限矩阵草案）
- 角色定义：
  - 数据平台管理员 / 超级管理员（Admin）：有系统管理后台账号，人数少且受控。
  - 其他角色（数据工程师、BI、业务 Owner 等）：本期不直接登录该模块，通过工单/协作方式间接提出需求。
- 权限矩阵（MVP）：
  - Admin 角色权限如下：
    - 查看数据源列表 & 详情：是
    - 新增数据源：是
    - 编辑数据源（含连接配置变更）：是
    - 启用 / 禁用数据源：是
    - 删除（软删）：是
    - 发起连接测试：是
    - 查看审计记录 / 测试相关日志摘要：是
    - 查看敏感字段（仅脱敏形式）：是
  - 其他角色：无直接访问“数据源管理”模块权限。

2.4 主流程
- 流程 1：新增数据源
  - Admin 进入“数据源管理”列表 → 点击“新增数据源” → 选择数据源类型（数据库、MQ、文件、API 等） →
    填写基础信息（名称、业务域、责任人、环境 prod/非 prod、标签等） →
    填写连接参数（如主机、端口、库名、用户名、密码/Token 等） →
    可选点击“连接测试”验证配置 → 校验通过后保存 → 默认状态（启用或禁用）按治理要求约定（建议默认启用，具体可在 PRD 中再细化）。
- 流程 2：编辑数据源
  - 在列表中通过搜索/筛选找到目标数据源 → 进入详情页或编辑页面 →
    调整描述、责任人、分组/标签或连接配置等字段 →
    可选再次发起“连接测试”验证新配置 → 保存成功 →
    系统记录审计日志，包含变更人、变更时间以及关键字段的变更摘要。
- 流程 3：启用 / 禁用数据源
  - Admin 在列表或详情页中使用“启用/禁用”开关 →
    若数据源被下游任务引用，则给出风险提示（如“当前有 X 个任务正在使用该数据源”） →
    用户二次确认后执行状态切换 →
    状态更新到数据库并在审计日志中记录一条状态变更记录。
- 流程 4：删除数据源（软删）
  - 仅 Admin 可见“删除”操作 →
    点击删除前，系统检查该数据源是否被任务/作业引用 →
    若仍有引用，则建议阻止删除或需要强二次确认（根据治理策略在 PRD 中细化） →
    删除后数据源标记为“已删除/不可用”，不再允许新的任务绑定，但历史记录与审计记录仍保留。
- 流程 5：连接测试
  - 在详情页或编辑表单中点击“连接测试” →
    后端根据数据源类型构造连接（数据库直连、HTTP 调用、Kafka 客户端连接等） →
    在设定超时时间内返回成功或失败结果 →
    前端展示“成功 / 失败 + 摘要文案”，详细错误仅写入日志/审计 →
    日志/审计中记录测试发起人、时间、目标数据源标识、结果及错误摘要。
- 流程 6：分组 / 标签管理与筛选
  - Admin 进入列表页 →
    使用业务域、环境、数据源类型、标签、状态等条件组合筛选 →
    支持关键字搜索（按名称/描述） →
    结果分页展示，并在每行显示基础信息和“被引用数量”等关键指标。
- 流程 7：审计 / 日志查看（基础版）
  - 在数据源详情页或统一“审计”入口中 →
    以数据源为主维度查看其配置变更历史、敏感字段访问行为（查看/导出） →
    若接入平台审计系统，可跳转或查询连接测试相关操作的日志记录。

2.5 边界 / 异常场景（≥ 11 条）
1. 删除 / 禁用被引用中的数据源
   - 描述：数据源已被采集任务 / 元数据作业引用时，Admin 尝试删除或禁用。
   - 期望：
     - 删除前强制校验引用情况，若存在引用，默认阻止删除，并提示需先迁移或解绑。
     - 禁用前给出明显风险提示，并要求二次确认。
2. 连接测试失败（网络不可达 / 超时 / 认证失败）
   - 描述：网络故障、目标系统不可达、认证凭证错误等导致测试失败。
   - 期望：前端仅展示统一失败结构 + 友好文案，详细错误记录在日志中；调用有合理超时策略。
3. 数据源凭证失效（密码修改、证书过期）
   - 描述：外部系统修改密码或证书，导致原配置失效。
   - 期望：连接测试失败时提示认证失败，便于管理员快速定位是“凭证问题”；日志中保留足够信息协助排障。
4. 重复数据源（同一系统、同一环境下多条近似配置）
   - 描述：新增时名称/主机/库名等高度相似。
   - 期望：系统做弱校验或提醒“可能已存在相似数据源”，鼓励复用/合并，但不强制阻止。
5. 并发编辑冲突
   - 描述：两名 Admin 同时编辑同一数据源并保存。
   - 期望：采用乐观锁或更新时间校验避免后保存覆盖先保存，给出“数据已被他人修改，请刷新后重试”的提示。
6. 敏感字段展示与修改
   - 描述：编辑页面展示密码/Token 等敏感字段时的行为。
   - 期望：仅以掩码形式展示，不在前端回显明文；如未修改则不更新后端存储，防止误清空。
7. 非生产环境配置错误（误连生产）
   - 描述：误将生产库配置到“非生产”环境标签。
   - 期望：通过命名规范、域名约定等做弱校验并提示风险（如“目标地址看起来像生产环境，请再次确认”），但不强制阻止。
8. 大量数据源下的分页与查询性能
   - 描述：数据源规模上千甚至上万时。
   - 期望：列表分页 + 筛选响应时间满足性能指标（见 2.6），避免一次性加载全部。
9. 不支持 / 未实现的数据源类型
   - 描述：用户尝试创建 ClickHouse 等未在白名单中的类型。
   - 期望：类型选择下拉仅展示受支持类型，对于未来计划支持的类型在文档/帮助中注明路线，不在 UI 中暴露。
10. 连接测试被频繁触发 / 滥用
    - 描述：同一数据源被高频连续测试，可能对目标系统造成额外负载。
    - 期望：后端增加简单限流/节流（如同一用户、同一数据源单位时间内限制次数），超出时返回提示“操作过于频繁，请稍后再试”。
11. 软删后的引用查询
    - 描述：某数据源被软删后，仍出现在历史任务记录中。
    - 期望：台账中标记为“已删除/不可用”，不再允许新的任务绑定，但历史任务记录仍可查询，并能看到该数据源曾被使用。

2.6 非功能需求
- 性能
  - 在 1,000 条数据源规模下，常用筛选 + 分页查询 P95 响应时间 ≤ 3 秒。
  - 单次连接测试在网络正常情况下，P95 响应时间 ≤ 10 秒，超时有明确提示。
- 安全
  - 密码 / Token / 密钥字段采用加密存储（如对称加密，密钥由安全团队统一管理），数据库中不落明文。
  - 所有写操作需登录态并通过统一鉴权机制，防止越权访问与 CSRF 等攻击。
- 审计
  - 新增/编辑/删除数据源、敏感字段查看/导出操作记录到审计系统（含操作人、时间、目标对象、操作类型、关键信息摘要）。
  - 连接测试操作写入日志/审计，可按数据源或时间范围进行检索追溯。
- 可观测性
  - 关键接口（CRUD、启用/禁用、连接测试等）有结构化日志，包含请求参数摘要、结果与耗时。
  - 对外部系统调用的失败/超时有明确的错误码和日志字段，便于监控和告警。


3) 关键决策与风险

3.1 关键决策与已选方案
- 敏感字段展示策略：
  - 决策：永远不在前端展示明文密码/Token/密钥，仅展示脱敏形式；后端加密存储。
- 错误信息展示粒度：
  - 决策：前端仅展示简要错误码/友好文案，详细错误信息（第三方报错、堆栈等）仅写入日志/审计。
- 状态模型简化：
  - 决策：仅保留“启用/禁用”两种状态，不引入“草稿/废弃”等复杂生命周期。
- 下游引用展示：
  - 决策：列表展示“被引用数量”，详情页展示引用任务/作业列表（只读），删除/禁用前提供引用情况提示。
- 连接测试记录策略：
  - 决策：不中台内建“连接测试记录表”，仅依赖统一日志/审计系统实现“可追溯”，不在业务界面提供测试历史列表。

3.2 风险分析
- 安全 & 合规风险
  - 若加密实现或密钥管理不当，仍可能存在敏感信息泄露风险，需要安全团队参与方案评审。
  - 错误信息若处理不严谨，仍可能通过日志或接口返回暴露内部细节。
- 操作风险
  - 少量 Admin 拥有全量操作能力（启用/禁用/删除），误操作可能导致生产任务大面积失败，需要操作流程规范与审计约束。
- 一致性风险
  - 若任务调度/元数据系统未完全对接本数据源台账，可能导致“台账状态”与“实际使用状态”不一致，需要与相关系统负责人对齐接口与约束策略。
- 性能与稳定性风险
  - 若连接测试实现不当（尤其是 Hive/Kafka/大数据组件），可能对目标系统造成额外压力，需要在实现与测试阶段进行压测与限流设计。

3.3 需要拍板/参与评审的角色
- 安全/合规负责人：
  - 拍板敏感字段加密方案、密钥管理模式、日志与审计中错误信息粒度，以及日志保留周期是否满足合规要求。
- 数据治理负责人 / 数据平台 Owner：
  - 拍板删除/禁用策略（是否允许在有引用时强制删除）、默认启用/禁用策略，以及未来生命周期状态扩展方向。
- 采集/调度系统负责人：
  - 拍板与调度/采集系统的对接边界：是否强依赖本模块的状态与配置，引用校验与提示程度如何实现。
