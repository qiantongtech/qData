# /prd-generate
基于澄清结论生成“功能级PRD”（不写代码）。

## 必须引用的上下文
- @docs/features/FEAT-xxx/01-clarification.md

## 输出（写入 docs/features/FEAT-xxx/02-prd.md）
PRD结构：
1. 背景与问题
2. 目标与成功指标（指标口径清楚）
3. 非目标（明确不做什么）
4. 用户与场景（用户故事）
5. 功能需求（编号、优先级、描述、验收标准）
6. 数据与权限（字段口径、权限矩阵、审计要求）
7. 交互与异常（概要，细节在文本原型）
8. 依赖与风险
9. 发布策略（灰度/回滚/监控）
10. 开发与测试拆解（可分派任务列表）

## 约束
- 每条功能需求必须有“可测试”的验收标准。
- 不确定处列【待确认项】。

## Gate（必须在输出末尾给出）
- 产物清单：写入 docs/features/FEAT-xxx/02-prd.md
- 验收清单（Checklist）：
    1) 每条需求都有验收标准
    2) 非目标明确
    3) 数据与权限写清（至少到字段/规则粒度）
    4) 风险与依赖齐全
    5) 发布策略最少说明：是否灰度/是否回滚/监控点
- Gate 判定：
    - READY_FOR_NEXT：以上全部满足
    - 否则 NOT_READY：列阻塞项（缺哪个章节/哪条规则未定）
- 下一步建议：
    - READY_FOR_NEXT -> /prototype-text（需引用 @docs/features/FEAT-xxx/02-prd.md）
