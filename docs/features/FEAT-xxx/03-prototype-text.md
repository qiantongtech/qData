## 页面 P1：数据源管理列表页（左树右表）

### 1. 页面名称 & 入口
- **页面名称**：数据源管理列表
- **入口**：
  - 数据中台管理后台左侧主菜单 `数据源管理`。
  - 点击菜单后进入本页，默认展示左侧树 + 右侧列表的主视图。

### 2. 适用角色/权限
- **角色范围**：
  - 仅 `Admin（数据平台管理员 / 超级管理员）` 可访问本页面。
  - 其他角色（数据工程师、BI、业务 Owner 等）无直接访问权限，通过工单间接处理。
- **权限说明**：
  - 只要能进入本模块的 Admin，即默认拥有本页面全部操作权限：
    - 查看列表、筛选/搜索；
    - 新增数据源；
    - 编辑数据源；
    - 启用 / 禁用数据源；
    - 删除（软删）数据源；
    - 发起连接测试；
    - 进入详情页查看引用与审计概要。

### 3. 页面结构（区块划分）
- **整体布局**：左侧树 + 右侧列表的经典管理后台布局，沿用现有系统设计风格。
  1. **左侧区块：分组树（导航）**
     - 形式：垂直树形/分组导航，固定宽度，随页面高度滚动。
     - 分组逻辑（示例，可根据实际字典配置）：
       - 第一层：业务域（如「用户域」「订单域」「营销域」等）；
       - 第二层（可选）：按环境拆分 `prod / 非 prod`；
       - 叶子节点：按数据源类型或标签聚合（如「MySQL」「Hive」「Kafka」等）。
     - 左侧树节点点击行为：
       - 高亮当前节点；
       - 右侧列表自动按对应维度进行筛选（例如点击「用户域 / prod / MySQL」）。
  2. **右侧上部：筛选区 + 顶部操作区**
     - 筛选区（横向排列，支持折行）：
       - 下拉 / 多选：业务域、环境（prod / 非 prod）、数据源类型、状态（启用/禁用）、标签；
       - 文本输入：关键字搜索（按名称 / 描述模糊匹配）。
     - 顶部操作按钮区（靠右对齐）：
       - `新增数据源` 主按钮（高亮），不提供批量操作按钮。
  3. **右侧中部：数据源表格列表**
     - 标准表格样式，支持列宽拖拽、悬浮提示等通用能力（沿用现有组件）。
     - 行点击或操作列按钮可进入详情 / 编辑 / 启用禁用 / 删除 / 连接测试。
  4. **右侧底部：分页区**
     - 位于表格底部右下侧，展示分页信息、页码切换和每页条数选择。

### 4. 字段清单

#### 4.1 筛选区字段
- **业务域（biz_domain）**
  - 类型：多选下拉（枚举）
  - 默认值：全部（不限制）
  - 校验：可为空；选择项必须来自预设字典
  - 是否展示：是
  - 是否可编辑：可编辑（筛选条件）

- **环境（env）**
  - 类型：单选下拉（枚举：`prod` / `non_prod`）
  - 默认值：全部（不限制）
  - 校验：可为空；若选择则必须为上述两项之一
  - 是否展示：是
  - 是否可编辑：可编辑（筛选条件）

- **数据源类型（type）**
  - 类型：多选下拉（枚举：mysql、postgresql、oracle、sqlserver、hive、kafka、ftp、sftp、http_api、doris）
  - 默认值：全部（不限制）
  - 校验：可为空；选择项必须来自受支持类型列表
  - 是否展示：是
  - 是否可编辑：可编辑（筛选条件）

- **状态（status）**
  - 类型：单选下拉（枚举：enabled / disabled）
  - 默认值：全部（不限制）
  - 校验：可为空
  - 是否展示：是
  - 是否可编辑：可编辑（筛选条件）

- **标签（tags）**
  - 类型：多选下拉（标签字典）
  - 默认值：全部（不限制）
  - 校验：可为空
  - 是否展示：是
  - 是否可编辑：可编辑（筛选条件）

- **关键字（keyword）**
  - 类型：文本输入
  - 默认值：空字符串
  - 作用：对名称 / 描述进行模糊匹配
  - 校验：允许空；长度上限按系统统一规则（如 100 字符）
  - 是否展示：是
  - 是否可编辑：可编辑（筛选条件）

#### 4.2 表格列字段
- **名称（name）**
  - 类型：字符串
  - 默认值：无，创建时必填
  - 校验：非空；长度 1–64；同一环境 + 类型内建议唯一（可做弱校验）
  - 是否展示：是（必展示列，可以点击进入详情）
  - 是否可编辑：在编辑页面可改

- **数据源类型（type）**
  - 类型：枚举（同上）
  - 默认值：创建时必选
  - 校验：非空；必须为受支持类型
  - 是否展示：是
  - 是否可编辑：在编辑页面可改（修改可能影响连接配置字段集）

- **业务域（biz_domain）**
  - 类型：枚举
  - 默认值：创建时必选
  - 校验：非空
  - 是否展示：是
  - 是否可编辑：在编辑页面可改

- **环境（env）**
  - 类型：枚举：`prod` / `non_prod`
  - 默认值：创建时必选
  - 校验：非空；可针对明显“非生产却像生产”的配置给出弱校验提示
  - 是否展示：是
  - 是否可编辑：在编辑页面可改

- **责任人（owner）**
  - 类型：字符串或用户选择器
  - 默认值：可为空或默认当前操作人（视平台统一规则）
  - 校验：长度限制，如 ≤ 64 字符
  - 是否展示：是
  - 是否可编辑：在编辑页面可改

- **状态（status）**
  - 类型：枚举：enabled / disabled
  - 默认值：建议新建后默认 `enabled`（可在实现时根据治理策略再确认）
  - 校验：非空
  - 是否展示：是（以状态标签形式展示）
  - 是否可编辑：通过启用/禁用操作修改

- **被引用数量（ref_count）**
  - 类型：整数
  - 默认值：0
  - 校验：≥ 0；允许通过异步任务刷新
  - 是否展示：是
  - 是否可编辑：否（只读，由系统统计）

- **标签（tags）**
  - 类型：字符串数组
  - 默认值：空数组
  - 校验：每个标签长度限制，如 ≤ 32 字符
  - 是否展示：是（以标签气泡形式）
  - 是否可编辑：在编辑页面可改

- **创建时间（created_at）**
  - 类型：时间戳
  - 默认值：系统生成
  - 校验：必填，由系统写入
  - 是否展示：是
  - 是否可编辑：否

- **更新时间（updated_at）**
  - 类型：时间戳
  - 默认值：系统生成
  - 校验：必填，由系统写入
  - 是否展示：是
  - 是否可编辑：否

- **操作列（actions）**
  - 类型：操作按钮集合
  - 默认值：N/A
  - 校验：N/A
  - 是否展示：是
  - 是否可编辑：否（但按钮启用状态随业务变化）

### 5. 交互与按钮 enable/disable 规则

#### 5.1 顶部操作
- **查询 / 筛选**
  - 行为：
    - 修改任一筛选条件后，点击 `查询` 按钮触发请求；
    - 可选提供 `重置` 按钮，一键清空所有筛选条件并重新加载默认列表。
  - `查询` 按钮：
    - Enabled 条件：始终可点（页面加载完成后）；
    - Disabled 条件：接口请求进行中可进入 loading + 禁用，防止重复触发；
    - 点击后：触发列表数据重新加载，展示加载态。
  - `重置` 按钮：
    - Enabled 条件：至少有一个筛选条件非默认值时可点击；
    - Disabled 条件：所有条件已是默认值；
    - 点击后：重置筛选表单为默认值，并自动触发一次查询。

- **新增数据源（primary 按钮）**
  - Enabled 条件：
    - 当前用户为 Admin 且页面加载完成；
    - 与列表数据无直接依赖，不因列表为空或加载中被禁用。
  - Disabled 条件：
    - 页面正在整体处于无权限态或严重错误态时可统一禁用所有操作；
  - 点击结果：
    - 打开 P2 新增数据源弹窗，表单默认空白。

#### 5.2 行级操作按钮
针对每行数据源，操作列包含：`编辑`、`详情`、`启用/禁用`、`删除`、`连接测试`。

- **编辑**
  - Enabled 条件：
    - 记录 `status` 为 enabled 或 disabled；
    - 记录未软删；
  - Disabled 条件：
    - 记录已软删（若在列表展示，操作列整体禁用或不展示）；
  - 点击结果：
    - 打开 P2 编辑弹窗，并填充当前数据源的基础信息与连接配置（敏感字段脱敏）。

- **详情**
  - Enabled 条件：
    - 同编辑按钮；
  - 点击结果：
    - 跳转至 P3 详情页。

- **启用 / 禁用**
  - 按钮文案：
    - 当前为 enabled → 操作按钮显示为 `禁用`；
    - 当前为 disabled → 操作按钮显示为 `启用`。
  - Enabled 条件：
    - 记录未软删；
  - Disabled 条件：
    - 记录已软删或当前接口请求进行中（进入 loading 态）；
  - 交互流程（禁用）：
    1. 点击 `禁用` → 调用接口前弹出二次确认弹窗；
    2. 若 `ref_count > 0`，弹窗文案强调风险：“当前有 X 个任务正在使用该数据源，禁用可能影响任务运行，请确认已评估影响后再操作。”；
    3. 用户选择“确认”后，按钮进入 loading 并禁用再次点击，调用后端接口；
    4. 成功：状态更新为 disabled，展示“禁用成功”提示；
    5. 失败：保持原状态，展示错误提示。
  - 交互流程（启用）类似，但提示文案可较为简单。

- **删除（软删）**
  - Enabled 条件：
    - 记录未软删；
  - Disabled 条件：
    - 记录已软删；
  - 交互流程：
    1. 点击 `删除` → 前端先请求后端校验引用情况；
    2. 若存在引用（ref_count > 0）：
       - 默认阻止删除，弹出提示：“当前有 X 个任务正在使用该数据源，请先迁移或解绑引用后再删除。”；
       - 不提供强制删除入口（可在后续治理策略中再讨论是否开放）。
    3. 若无引用：
       - 弹出确认弹窗，提示“删除为软删操作，删除后将不再作为可用数据源展示，但历史记录仍保留，确认继续吗？”；
       - 用户确认后按钮进入 loading，调用后端软删接口；
       - 成功：该行从列表中移除或标记为“已删除/不可用”并隐藏操作按钮；展示“删除成功”提示；
       - 失败：提示错误信息。

- **连接测试**
  - Enabled 条件：
    - 记录状态为 enabled 或 disabled（软删后不提供测试）；
    - 该数据源的配置模型在后端支持连接测试；
  - Disabled 条件：
    - 记录已软删；
    - 当前存在该行连接测试请求进行中（防止重复点击）。
  - 交互流程：
    1. 点击 `连接测试` → 按钮进入 loading + 禁用；
    2. 后端在设定超时时间内完成测试；
    3. 成功：以全局提示条或行内提示形式展示“连接测试成功（耗时 X ms）”；
    4. 失败：展示统一友好文案，如：“连接测试失败，请检查主机地址/端口/账号密码等配置。如仍有问题，请联系运维人员。（错误码：XXX）”；
    5. 完成后结束 loading，按钮恢复可点击。

### 6. 列表规则
- **筛选规则**
  - 所有筛选条件支持组合筛选（业务域 + 环境 + 类型 + 状态 + 标签 + 关键字）。
  - 改变筛选条件不会立即触发请求，需点击 `查询` 按钮（可根据统一规范调整为自动查询）。

- **排序规则**
  - 默认排序：
    - 按 `创建时间（created_at）` 倒序，最新创建的数据源排在最前（与澄清选择一致）。
  - 支持切换排序字段：
    - 至少支持 `创建时间`、`更新时间` 两个字段升/降序切换；
    - 可选支持按 `名称` 字典序升序排序。
  - 列头点击排序图标即可切换。

- **分页规则**
  - 默认每页条数：10 条；
  - 可选条数：10 / 20 / 50；
  - 最大允许选择：50 条/页；
  - 翻页操作（上一页 / 下一页 / 指定页码）触发重新请求当前条件下的数据。

- **批量操作**
  - 本期 **不提供任何批量操作**（不支持批量启用/禁用、批量删除），所有操作逐条进行，以降低误操作风险。

### 7. 状态（加载/空/错/无权限/成功）
- **加载态**
  - 触发时机：
    - 页面首次进入时；
    - 点击查询 / 重置 / 翻页 / 更改每页条数时；
  - 展现形式：
    - 表格区域展示骨架屏或 loading 动画，隐藏旧数据；
    - 顶部操作按钮（新增等）保持可用。

- **空态**
  - 场景：
    - 系统无任何数据源记录；
    - 当前筛选条件下查询结果为空。
  - 展现形式：
    - 表格区域展示空态插画或图标 + 文案；
    - 示例文案：
      - 全局无数据时：“暂未配置任何数据源，请点击右上角‘新增数据源’进行配置。”
      - 筛选结果为空时：“未找到符合当前筛选条件的数据源，请尝试调整筛选条件。”
    - `新增数据源` 按钮正常可用。

- **错误态**
  - 场景：列表接口调用失败（网络异常、服务错误等）。
  - 展现形式：
    - 表格区域展示统一错误提示；
    - 示例文案：“数据源列表加载失败，请稍后重试。如多次失败，请联系运维人员。”
    - 可提供 `重试` 按钮再次发起请求。

- **无权限态**
  - 场景：用户虽登录管理后台，但不具备访问数据源管理模块权限。
  - 展现形式：
    - 页面整体展示无权限提示，不展示左树右表的业务内容；
    - 示例文案：“当前账号暂无访问数据源管理模块的权限，如需开通请联系系统管理员。”

- **成功态**
  - 场景：新增 / 编辑 / 启用 / 禁用 / 删除 / 连接测试成功后。
  - 展现形式：
    - 顶部或右下角统一成功提示条（如 “新增数据源成功”、“删除成功”、“连接测试成功” 等）；
    - 列表自动刷新最新数据（对删除/状态变更等操作）。

### 8. 可测试验收点（≥ 8 条）
1. 在无任何筛选条件时，进入列表页默认展示最近创建的数据源，排序为按创建时间倒序。
2. 修改业务域 + 环境 + 类型 + 状态 + 标签 + 关键字任意组合后，点击 `查询`，列表仅展示满足所有条件的数据源。
3. 点击 `重置` 按钮后，所有筛选条件恢复默认值，并自动刷新列表为默认视图。
4. 列表页右上角 `新增数据源` 按钮在页面加载成功后始终可点击，点击后弹出新增表单。
5. 对状态为 enabled 的数据源行，点击 `禁用`，若 `ref_count > 0`，弹出二次确认且展示引用数量，确认后状态更新为 disabled 并有成功提示。
6. 对无下游引用的数据源行，点击 `删除`，弹出软删确认弹窗；确认后该数据源从列表消失且无法再作为可用数据源出现在下拉选择中。
7. 在网络或后端故障场景下，加载列表接口失败时，表格展示错误文案和 `重试` 按钮，点击 `重试` 后在服务恢复时可以正常加载数据。
8. 在无任何数据源记录的情况下进入列表页，表格区域展示空态文案“暂未配置任何数据源，请点击右上角‘新增数据源’进行配置。”，且 `新增数据源` 按钮可用。
9. 对任意一行点击 `连接测试`，在配置正确情况下，多次测试均返回“连接测试成功（耗时 X ms）”文案，且按钮在测试过程中处于 loading 禁用状态。
10. 对同一数据源在短时间内高频点击 `连接测试`，当后端触发限流时，前端收到统一“操作过于频繁，请稍后再试”提示。

## 页面 P2：新增 / 编辑数据源弹窗（Dialog）

### 1. 页面名称 & 入口
- **页面名称**：新增 / 编辑数据源（弹窗）
- **入口**：
  - 新增：
    - 列表页右上角 `新增数据源` 按钮；
  - 编辑：
    - 列表行操作列 `编辑` 按钮；
    - 详情页顶部 `编辑` 按钮。

### 2. 适用角色/权限
- 仅 Admin 角色可打开本弹窗。
- 进入弹窗前已完成模块级别权限校验；弹窗内部不再做额外权限分级。

### 3. 页面结构（区块划分）
- **整体形态**：居中 Dialog 弹窗，宽度适中（与现有表单类弹窗保持一致），可垂直滚动。
- **区块划分**：
  1. 标题区：展示「新增数据源」或「编辑数据源」标题。
  2. 表单区上半部分：**基础信息** 区块。
  3. 表单区下半部分：**连接配置** 区块（字段随数据源类型变化）。
  4. 底部操作区：从左到右依次为 `连接测试`（次按钮）、`取消`、`保存`（主按钮）。

### 4. 字段清单

#### 4.1 基础信息区
- **数据源名称（name）**
  - 类型：字符串输入框
  - 默认值：
    - 新增：空；
    - 编辑：填充当前名称。
  - 校验：
    - 必填，非空；
    - 长度 1–64；
    - 建议在同一环境 + 类型下唯一，如存在相同名称可给出提示但不强制阻止。
  - 是否展示：是
  - 是否可编辑：新增/编辑均可编辑。

- **描述（desc）**
  - 类型：多行文本
  - 默认值：新增为空；编辑填充当前描述
  - 校验：长度上限，例如 0–256 字符
  - 是否展示：是
  - 是否可编辑：是

- **业务域（biz_domain）**
  - 类型：下拉单选
  - 默认值：新增为空或预选某默认业务域；编辑填充当前值
  - 校验：必填；必须从预设字典中选择
  - 是否展示：是
  - 是否可编辑：是

- **环境（env）**
  - 类型：下拉单选（prod / non_prod）
  - 默认值：新增默认为 non_prod（可按治理策略调整）；编辑填充当前值
  - 校验：
    - 必填；
    - 若选择 non_prod 但主机/库名等看起来像生产环境，可给出弱提示但不强制阻止。
  - 是否展示：是
  - 是否可编辑：是

- **责任人（owner）**
  - 类型：用户选择器或文本
  - 默认值：可默认当前登录用户；编辑填充当前值
  - 校验：非必填；长度 ≤ 64 字符
  - 是否展示：是
  - 是否可编辑：是

- **标签（tags）**
  - 类型：多选标签组件
  - 默认值：空数组；编辑为当前标签集合
  - 校验：单个标签长度 ≤ 32 字符
  - 是否展示：是
  - 是否可编辑：是

- **数据源类型（type）**
  - 类型：下拉单选
  - 默认值：
    - 新增：空，需用户选择；
    - 编辑：显示当前类型，通常不建议修改（如允许修改需触发连接配置区字段集重置）。
  - 校验：必填；必须为受支持类型之一
  - 是否展示：是
  - 是否可编辑：
    - 新增：可编辑；
    - 编辑：一般禁用修改（建议），或仅在满足条件时允许修改并提示风险。

#### 4.2 连接配置区（以数据库类为示例）
- **主机（host）**
  - 类型：字符串输入
  - 默认值：新增为空；编辑填充当前配置
  - 校验：必填；应为合法主机名或 IP
  - 是否展示：是
  - 是否可编辑：是

- **端口（port）**
  - 类型：数字输入
  - 默认值：可按类型默认（如 MySQL 默认 3306）
  - 校验：
    - 必填；
    - 必须为整数，范围如 1–65535
  - 是否展示：是
  - 是否可编辑：是

- **数据库名（database）**
  - 类型：字符串输入
  - 默认值：新增为空；编辑填充当前值
  - 校验：必填；格式按数据库命名规则
  - 是否展示：是
  - 是否可编辑：是

- **用户名（username）**
  - 类型：字符串输入
  - 默认值：新增为空；编辑填充当前值
  - 校验：必填
  - 是否展示：是
  - 是否可编辑：是

- **密码（password，前端脱敏展示）**
  - 类型：密码输入框
  - 默认值：
    - 新增：空；
    - 编辑：以固定掩码（如 `********`）显示，代表已存在密码。
  - 校验：
    - 新增：必填；
    - 编辑：若用户不修改，则不覆盖后端原有加密值；若用户输入新密码，则校验非空并更新。
  - 是否展示：是（脱敏）
  - 是否可编辑：是

- **额外参数（parameters）**
  - 类型：键值对或 JSON 文本
  - 默认值：空
  - 校验：按实现约定进行基本格式校验
  - 是否展示：是
  - 是否可编辑：是

> 对 Kafka、FTP/SFTP、HTTP API 等类型，可在同一区块定义对应字段（如 `bootstrap_servers`、`topic`、`base_path`、`base_url`、`auth_type` 等），本文字原型不一一展开，前端需根据类型动态展示。

### 5. 交互与按钮 enable/disable 规则

- **连接测试按钮**
  - 文案：`连接测试`
  - Enabled 条件：
    - 至少满足一组最小连接配置（如 host + port + database + username + password 均已填且通过基础校验）；
  - Disabled 条件：
    - 必填连接字段未填写完整或校验失败；
    - 当前已有连接测试请求进行中（loading 中）。
  - 点击行为：
    1. 按钮进入 loading 状态并禁用；
    2. 调用后端连接测试接口；
    3. 成功：弹出或顶部展示“连接测试成功（耗时 X ms）”提示；
    4. 失败：展示友好错误文案及错误码，不暴露内部堆栈；
    5. 完成后，按钮恢复可点击。

- **保存按钮**
  - 文案：`保存`
  - Enabled 条件：
    - 所有必填字段（基础信息 + 连接配置）均通过前端校验；
    - 不依赖连接测试是否成功（本期不强制先测试再保存）。
  - Disabled 条件：
    - 存在未通过校验的必填字段；
    - 保存请求进行中（loading）。
  - 点击行为：
    1. 触发表单整体校验，若有错误，阻止提交并在对应字段下方展示红色错误提示；
    2. 校验通过后，按钮进入 loading 状态并禁用，调用保存接口；
    3. 成功：
       - 新增：关闭弹窗，列表刷新并高亮或滚动到新建记录；展示“新增数据源成功”；
       - 编辑：关闭弹窗，列表/详情刷新并展示“保存成功”；
    4. 失败：保持弹窗打开，展示统一错误提示“保存失败，请稍后重试。如多次失败，请联系运维人员。”，按钮退出 loading。

- **取消按钮 / 关闭图标**
  - Enabled 条件：始终可用（保存/连接测试 loading 时是否允许关闭可按平台统一规范，一般允许）。
  - 点击行为：
    - 关闭弹窗；
    - 若存在未保存修改，弹出“确认放弃当前未保存修改？”二次确认，避免误关。

### 6. 列表规则（不适用）
- 本页面为弹窗表单，没有独立列表；连接配置内如有子列表（如多个 Kafka topic 等），可按通用列表规则设计，不在本版本详细展开。

### 7. 状态（加载/空/错/无权限/成功）
- **加载态**
  - 新增模式：弹窗打开即为可编辑态，不需要额外加载；
  - 编辑模式：弹窗打开时若需从后端拉取最新详情，在数据未返回前展示表单骨架或整体 loading 状态，禁用保存和连接测试。

- **空态**
  - 对应字段为空时，表单正常展示；不做特殊空态页面。

- **错误态**
  - 字段级错误：在字段下方展示具体校验错误文案，如“必填项”、“格式不正确”、“端口必须为数字”等；
  - 接口错误：保存 / 连接测试接口失败时，在弹窗顶部或底部展示统一错误提示。

- **无权限态**
  - 非 Admin 用户无法进入本弹窗；一旦触发入口，由上游路由/权限层拦截，不在弹窗内单独处理。

- **成功态**
  - 连接测试成功：展示成功提示；
  - 保存成功：关闭弹窗并在父页面展示“新增/保存成功”。

### 8. 可测试验收点（≥ 8 条）
1. 在新增模式下，未填写必填字段（如名称、业务域、类型、host、port、database、username、password）时，点击 `保存` 按钮，表单阻止提交并在对应字段下展示提示。
2. 在新增模式下，仅填写部分连接字段但未满足最小配置时，`连接测试` 按钮处于禁用状态。
3. 在新增模式下，填写完整有效的数据库连接信息后，点击 `连接测试`，若后端连通，前端展示“连接测试成功（耗时 X ms）”提示。
4. 在新增模式下，故意填写错误密码，点击 `连接测试`，前端展示统一的失败文案和错误码，不展示堆栈。
5. 在编辑模式下，对敏感字段不做任何修改直接保存，后端保留原加密密码，前端不传空值（可通过抓包或日志验证）。
6. 在编辑模式下，修改描述/业务域/环境等非敏感字段并保存，重新打开弹窗显示为新值。
7. 在编辑模式下，模拟并发编辑（后保存的一方版本号落后），点击 `保存` 时得到“数据已被他人修改，请刷新后重试”的提示，保存失败。
8. 在任一模式下，点击右上角关闭或 `取消` 按钮且当前有未保存修改时，弹出二次确认提示；选择“取消”则停留在弹窗，选择“确认离开”则关闭弹窗并丢弃修改。
9. 在连接测试或保存进行中，按钮处于 loading 状态且无法重复点击，接口返回后恢复正常。

## 页面 P3：数据源详情页

### 1. 页面名称 & 入口
- **页面名称**：数据源详情
- **入口**：
  - 列表页行操作列 `详情` 按钮；
  - 列表中名称列可点击，跳转至详情页。

### 2. 适用角色/权限
- 仅 Admin 角色可访问详情页。
- 无额外细粒度权限控制，进入详情页后可执行编辑、启用/禁用、删除、连接测试等操作。

### 3. 页面结构（区块划分）
- **整体布局**：单页布局，保留左侧主菜单，右侧为详情内容区域。
  1. 顶部信息条：
     - 显示数据源名称、类型、环境、状态等关键字段；
     - 右上角展示操作按钮：`编辑`、`启用/禁用`、`删除`、`连接测试`。
  2. 基础信息区块：
     - 以卡片或表格形式展示：名称、描述、业务域、环境、责任人、标签、创建时间、创建人、更新时间、更新人、当前状态等。
  3. 连接配置概要区块：
     - 展示连接相关参数（host、port、database、username 等）；敏感字段（password、token 等）仅以脱敏形式（如 `********`）展示。
     - 全部为只读。
  4. 下游引用信息区块：
     - 展示被引用数量；
     - 列表展示被哪些任务/作业引用（任务名称、所属系统/项目、环境等，只读）。
  5. 审计概要区块：
     - 展示近期若干条配置变更摘要、敏感字段访问记录摘要（如“某时间由某人编辑了连接配置”等）。

### 4. 字段清单

#### 4.1 基础信息区字段
- **名称（name）**：同前，展示当前值，只读。
- **描述（desc）**：展示当前描述，只读。
- **业务域（biz_domain）**：展示为人类可读名称，只读。
- **环境（env）**：展示为「生产环境」/「非生产环境」等文案，只读。
- **责任人（owner）**：展示责任人姓名或标识，只读。
- **标签（tags）**：展示为多个标签气泡，只读。
- **状态（status）**：展示启用/禁用状态标签。
- **创建时间（created_at） / 创建人（created_by）**：只读。
- **更新时间（updated_at） / 更新人（updated_by）**：只读。

#### 4.2 连接配置概要区字段（以数据库类为例）
- **类型（type）**：展示枚举值对应的中文说明，如「MySQL 数据库」。
- **主机（host）**：只读展示。
- **端口（port）**：只读展示。
- **数据库名（database）**：只读展示。
- **用户名（username）**：只读展示。
- **密码（password 显示）**：固定展示掩码字符串，如 `********`，不提供明文查看。
- **额外参数（parameters）**：可折叠展示。

#### 4.3 下游引用信息区字段
- **被引用数量（ref_count）**：整数，只读。
- **引用任务列表（refs）**（表格）
  - 列字段：
    - 任务名称（job_name）：字符串；
    - 所属系统/项目（system）：字符串；
    - 环境（env）：枚举；
    - 其它必要标识字段（按后端接口定义）。

#### 4.4 审计概要区字段
- **审计记录列表（audit_logs）**（摘要）
  - 列字段：
    - 操作时间（operate_time）；
    - 操作人（operator_name）；
    - 操作类型（action_type，如 create/update/delete/secret_view 等）；
    - 变更摘要 / 结果摘要（change_summary / result_summary）。

### 5. 交互与按钮 enable/disable 规则

- **编辑按钮**
  - Enabled 条件：
    - 当前数据源未软删；
  - Disabled 条件：
    - 数据源已被标记为删除（若仍可访问详情）；此时按钮可隐藏或禁用；
  - 点击行为：
    - 打开 P2 编辑弹窗。

- **启用 / 禁用按钮**
  - 文案与交互规则与列表页一致：
    - 当前为 enabled → 按钮显示 `禁用`；
    - 当前为 disabled → 按钮显示 `启用`；
  - 禁用时如有引用，展示风险提示并要求二次确认；
  - 状态变更成功后，顶部状态标签和列表中的状态同步更新。

- **删除按钮**
  - 逻辑同列表页删除操作：
    - 若存在引用（ref_count > 0）：阻止删除并提示需先迁移/解绑；
    - 若无引用：弹确认框，确认后调用软删接口；
  - 删除成功后：
    - 可自动跳转回列表页，并在列表页展示“删除成功”的提示；
    - 或在详情页展示“该数据源已删除/不可用”，隐藏编辑/启用/禁用/删除按钮。

- **连接测试按钮**
  - Enabled 条件：
    - 当前数据源配置完整且未软删；
  - Disabled 条件：
    - 数据源已软删；
    - 测试进行中；
  - 点击行为：
    - 审核同列表行 `连接测试`，但在详情页更偏向于确认当前配置可用；
    - 成功/失败提示内容同前。

- **下游引用列表交互**
  - 行点击：默认不跳转；如后续集成，可支持跳转到任务详情，但本期可作为仅展示。

### 6. 列表规则
- 下游引用任务列表：
  - 支持简单分页（如每页 10 条），默认按任务名称或创建时间排序（可按后端约定）；
  - 不支持在此处直接编辑任务，仅查看。

### 7. 状态（加载/空/错/无权限/成功）
- **加载态**
  - 进入详情页时，在后端数据未返回前展示整体 loading 骨架；
  - 操作按钮（编辑/启用/禁用/删除/连接测试）在加载完成前禁用。

- **空态**
  - 对于下游引用列表为空时，展示文案“当前暂无下游任务引用该数据源。”；
  - 对于审计记录为空时，展示文案“暂无相关审计记录。”。

- **错误态**
  - 详情主接口失败时，整体展示错误信息：“详情加载失败，请稍后重试。”，并提供返回列表按钮；
  - 局部子模块（如引用列表、审计列表）加载失败时，可分别展示局部错误提示。

- **无权限态**
  - 非 Admin 用户访问详情路由时，由路由层拦截并展示无权限页，不展示详情内容。

- **成功态**
  - 在详情页内执行启用/禁用/删除/连接测试/编辑保存后，成功均通过统一成功提示告知用户；
  - 页面数据自动刷新以反映最新状态。

### 8. 可测试验收点（≥ 8 条）
1. 从列表页点击数据源名称或 `详情` 按钮，可成功跳转至详情页并展示与列表行一致的基础信息。
2. 在详情页顶部点击 `编辑` 按钮，可打开编辑弹窗，字段值与详情区一致；保存后返回详情页，基础信息与连接配置概要更新为最新值。
3. 在详情页点击 `禁用` 按钮，若数据源被下游任务引用，弹出风险提示并要求二次确认；确认后状态变为禁用，顶部状态标签更新，返回列表页查看同一数据源状态也为禁用。
4. 在详情页点击 `删除` 按钮且数据源存在引用时，禁止删除并展示“当前有 X 个任务正在使用该数据源，请先迁移或解绑引用后再删除。”。
5. 在详情页点击 `删除` 按钮且数据源无引用时，经确认后删除成功，自动返回列表页，并在列表中无法再找到该数据源。
6. 在详情页点击 `连接测试` 按钮，若实际配置可连通，前端展示“连接测试成功（耗时 X ms）”，同时在日志/审计中有一条对应记录。
7. 在详情页加载下游引用信息时，如无任何引用任务，引用列表区展示“当前暂无下游任务引用该数据源。”提示，并不显示错误。
8. 在详情页加载审计概要时，如已有配置变更和敏感字段访问记录，则审计列表展示包含时间、操作者、操作类型、摘要信息的记录，可按时间倒序排列。
